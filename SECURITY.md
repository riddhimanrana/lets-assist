# Security Policy

## Supported Versions

We actively maintain security fixes for the live application and the current development branch.

| Branch / release | Supported |
| ---------------- | --------- |
| `main` (production) | :white_check_mark: |
| `development` (active development) | :white_check_mark: |
| Older branches and archived releases | :x: |

## Reporting a Vulnerability

If you believe you have found a security vulnerability in Let's Assist, please **do not** open a public issue or pull request.

Instead, email [contact@lets-assist.com](mailto:contact@lets-assist.com) with:

- A clear description of the issue and the affected area
- Steps to reproduce
- Any relevant URLs, account roles, screenshots, logs, or proof of concept
- The impact you believe the issue has

We aim to:

- Acknowledge new reports within 2 business days
- Triage and confirm valid reports as quickly as possible
- Provide status updates while a fix is in progress
- Coordinate responsible disclosure before public release of details

If the issue involves active abuse or account compromise, include that in the subject line so we can prioritize it.

Please avoid tests that may disrupt service, access data you do not own, or violate privacy. Good-faith security research is appreciated.
